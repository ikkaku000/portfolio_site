# プロジェクト仕様書

## 1. プロジェクト概要
- 個人ポートフォリオWebサイトを構築し、インターネット上に公開する。
- 主な掲載対象は、AIを活用して制作した成果物（業務効率化ツール、自動化ツール、Webアプリ、デスクトップアプリなど）とする。
- 本ドキュメントは、実装前に要件を整理し、認識のズレを防ぐことを目的とする。

## 2. プロジェクト目的
- 制作実績を分かりやすく提示し、閲覧者が「何を作れる人か」を短時間で理解できる状態にする。
- 将来的な採用応募・案件獲得・自己紹介のいずれにも活用できる土台を作る。

## 3. 想定ユーザー
- 採用担当者
- エンジニア・デザイナーなどの技術職
- クライアント候補

## 4. スコープ
### 4.1 対象（In Scope）
- ポートフォリオサイトの設計・実装
- 作品情報の掲載
- サーバー公開（本番環境で第三者が閲覧可能な状態）

### 4.2 対象外（Out of Scope）
- 公開後の継続的なマーケティング施策（SNS運用や広告運用）
- 大規模CMSの新規開発

## 5. 機能要件（初版）
### 5.1 必須ページ
- トップページ
- 作品一覧ページ
- 作品詳細ページ
- プロフィール（About）ページ
- 連絡先（Contact）ページ

### 5.2 作品情報の必須項目
- タイトル
- 概要
- 制作背景 / 解決した課題
- 使用技術
- 担当範囲
- 成果（数値があれば記載）
- 画像（スクリーンショット）
- 関連リンク（GitHub、デモURLなど）

### 5.3 検討機能（任意）
- カテゴリ/タグでの絞り込み
- 日本語/英語の言語切替
- 管理しやすい更新方式（Markdown、CMS、Notion連携など）

## 6. 非機能要件（初版）
- レスポンシブ対応（PC / タブレット / スマートフォン）
- 一般的なSEO設定（title、description、OGP、構造化を可能な範囲で対応）
- ページ表示速度に配慮した実装（画像最適化、不要な処理の削減）
- 最低限のアクセシビリティ配慮（見出し構造、代替テキスト、キーボード操作）
- HTTPSで公開

## 7. デザイン方針
- 参考UI: https://hasegawahiroshi.jp/works
- 参考サイトは「雰囲気・情報の見せ方」を参考にし、文言・構成・ビジュアルは独自性を持たせる。
- シンプルで読みやすく、作品が主役になるデザインを優先する。

## 8. 公開要件
- 本番URLで外部からアクセス可能であること
- 主要ページ（トップ、作品一覧、作品詳細、About、Contact）が表示できること
- 各作品詳細ページに必要情報が欠けなく表示されること

## 9. 成果物
- 公開済みポートフォリオサイト
- 本仕様書（このファイル）
- 掲載作品データ（画像・説明文・リンク一式）

## 10. 決定事項
### 10.1 確定事項
- サイトの第一目的: 趣味として制作・公開する（自己表現 / 学習記録を主目的とする）
- ドメイン: いったん無料ドメインで公開し、必要に応じて後から独自ドメイン化する
- 問い合わせ導線: メール直リンクのみ
- 言語対応: 日本語のみ
- 守秘義務のある実績: 匿名化して掲載する（企業名・機微情報は非公開）

### 10.2 技術選定
- 技術スタック（フレームワーク）: Next.js + TypeScript
- ホスティング先: Vercel

## 11. 実装フェーズ（MVPタスク）
### 11.1 全体方針
- まずは「公開して見せられる最小構成（MVP）」を作る。
- 初版では必須ページの実装と公開を優先し、任意機能は後続フェーズで対応する。

### 11.2 フェーズ一覧
#### Phase 1: 開発環境の準備
- Next.js + TypeScriptの初期プロジェクトを作成する
- GitHubリポジトリを作成し、初回コミットを行う
- Vercelとの連携準備を行う
- 完了条件: ローカルで起動でき、GitHubに初期コードが反映されている

#### Phase 2: 基本レイアウトと共通UI
- ヘッダー / フッター / ナビゲーションを実装する
- 全ページ共通のレイアウトを作成する
- フォント、色、余白など最低限のデザイントークンを決める
- 完了条件: 全ページで共通レイアウトが適用され、画面崩れがない

#### Phase 3: 必須ページの実装
- トップページを実装する
- 作品一覧ページを実装する
- 作品詳細ページを実装する（最低1件のサンプルデータ表示）
- AboutページとContactページ（メールリンク）を実装する
- 完了条件: 5.1の必須ページがすべて遷移・表示できる

#### Phase 4: 作品データの整備と反映
- 5.2の必須項目に沿って作品データを作成する
- 守秘義務対象の情報を匿名化した文面に調整する
- 作品カードと詳細ページに必要項目を反映する
- 完了条件: 各作品詳細ページに必須項目の欠落がない

#### Phase 5: 品質確認（公開前チェック）
- レスポンシブ表示（PC / タブレット / スマホ）を確認する
- title / description / OGPなど基本SEOを設定する
- 画像の代替テキスト、見出し構造を確認する
- 完了条件: 6章・8章の要件を満たしている

#### Phase 6: 公開
- Vercelへデプロイする
- 本番URLで全ページ動作を最終確認する
- 必要に応じてREADMEに運用メモを残す
- 完了条件: 第三者が本番URLで閲覧可能

### 11.3 着手順（実行順）
- Step 1: Phase 1を完了させる（環境構築とリポジトリ連携）
- Step 2: Phase 2で共通レイアウトを先に固める
- Step 3: Phase 3で必須ページを最低限の見た目で実装する
- Step 4: Phase 4で作品データを投入し、内容を実運用レベルにする
- Step 5: Phase 5で公開前チェックを行い、不備を修正する
- Step 6: Phase 6でVercel公開し、本番確認で完了とする

### 11.4 MVP完了定義
- 必須5ページが本番URLで閲覧できる
- 作品詳細に必須項目が表示される
- Contactページからメールで連絡できる
- スマートフォン表示でも主要導線が機能する
